Interviewer: Great. Let’s move to SQL. Can you explain how window functions work and give an example?

Candidate: Sure. Window functions allow us to perform calculations across a set of rows related to the current row without collapsing the result into a single output like GROUP BY does.

For example, if we want to calculate the difference between consecutive rows in a time series, we can use the LAG function:

SELECT 
    user_id,
    event_time,
    event_value,
    event_value - LAG(event_value) OVER (PARTITION BY user_id ORDER BY event_time) AS diff
FROM events;

This helps analyze trends without losing row-level granularity.
